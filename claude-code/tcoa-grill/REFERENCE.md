# tcoa-grill 参考文档

## 一、简介

tcoa-grill 是 TCOA 需求流程的**可选前置环节**，用于逐项追问模糊需求，直到达成共识。追问完成后自动查询 business-rules-mapping，输出结构化的 `grill-result`，供下游 tcoa-context(init) 消费，实现**结合现有代码生成文档**。

## 二、核心价值

| 问题 | 解决方案 |
|------|----------|
| 用户需求模糊，边界不清 | 逐项追问，一次一个问题 |
| 生成的文档与现有代码脱节 | 自动查询 business-rules-mapping，填充代码上下文 |
| 重复查询浪费 token | grill-result 一次生成，下游直接消费 |

## 四、数据流

```
用户需求
    │
    ▼
┌─────────┐    追问达成共识    ┌─────────────────────┐
│  grill  │ ─────────────────▶ │ grill-result.json   │
└─────────┘                    └─────────────────────┘
                                         │
                                         ▼ 读取 + 删除
                               ┌─────────────────────┐
                               │ tcoa-context(init)  │
                               └─────────────────────┘
                                         │
                                         ▼ 填充模板
                               ┌─────────────────────┐
                               │ proposal.md         │
                               │ §代码上下文          │
                               └─────────────────────┘
                                         │
                                         ▼ 参考
                               ┌─────────────────────┐
                               │ tcoa-execute        │
                               │ → design.md         │
                               └─────────────────────┘
```

## 五、兜底机制

| 场景 | 处理 |
|------|------|
| 未走 grill | proposal.md §代码上下文 留空，tcoa-execute 的 GitNexus 预检兜底 |
| grill 未命中模块 | 输出 `bizCode: null`，提示"可能是新模块" |
| grill-result.json 过期（>1小时） | 视为不存在，走兜底 |

## 六、文件清单

| 文件 | 用途 |
|------|------|
| `.tcoa/grill-result.json` | grill 输出的临时文件，tcoa-context(init) 消费后删除 |
| `.tcoa/templates/proposal.md` | proposal.md 模板，含 §代码上下文 占位 |
| `business-rules-mapping/references/*.md` | 业务模块与代码的映射关系 |
