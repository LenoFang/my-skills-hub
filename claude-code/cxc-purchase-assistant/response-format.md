# 返回格式规范

> 本文件在 SKILL.md Step 10-11 加载。定义三模块的渲染模板 + 推荐提问硬规则。

---

## 1. 三模块统一结构

每次返回给企微的消息由三个模块拼装：

```
[AI 总结]           ← LLM #2 (Render) 生成
[数据块]             ← skill 宿主用模板预渲染，不进 LLM
💡 试试这样问： a | b | c   ← LLM #3 (Recommend) 精选
```

---

## 2. AI 总结（ai_summary）

### 约束

- **长度**：1-3 句
- **语气**：冷静、简洁、数据导向
- **禁止**：感叹号、情感词（"太棒了"、"恭喜"）、营销话术、emoji

### 按场景侧重

| 场景类型 | 总结重点 |
|---|---|
| 精确查单 | 当前状态、审批进度（到第几步/共几步）、关键时间节点 |
| 列表查询 | 总数量、状态分布（N 条审批中/N 条已完成）、金额合计、异常项提示 |
| 统计查询 | 总量、总金额、主要分布维度、同环比对比（如有） |
| 模糊匹配 | 匹配到的结果数量、各条简要状态，引导用户确认 |
| 无结果 | 建议换关键词，或提示可能相似的单据 |

### Render prompt 模板

```
你是采购数据摘要助手。根据查询结果生成 1-3 句中文摘要。

规则：
- 冷静简洁，数据导向
- 不加感叹号和情感词
- 不修改或编造任何数字（数字由数据卡片展示，你只做文字总结）
- 按当前场景类型侧重（见下方 scenario_focus）

scenario_type: {scenario_type}
scenario_focus: {focus_points}
user_input: {user_input}
data_summary: {data_summary}

输出纯文本，1-3 句。
```

## 4. 推荐提问（recommendations）

### 硬规则

| 规则 | 值 |
|---|---|
| 每条最大长度 | 15 字 |
| 语气 | 口语化，无标点 |
| 数量 | 2-4 条 |
| 优先级排序 | 异常项 > 金额排序 > 分布下钻 > 同环比 > 明细下钻 |
| 禁止 | 开放式问题、重复用户已问过的问题 |
| 展示格式 | `💡 试试这样问： {a} \| {b} \| {c}` |

### Recommend prompt 模板

```
你是采购助手的推荐提问生成器。从候选池中选 2-4 条最有价值的推荐提问。

规则：
- 每条不超过 15 字
- 口语化，不加标点
- 不重复用户本轮已问的内容
- 不生成开放式问题（如"还有什么想看的"）
- 按优先级：异常项 > 金额排序 > 分布下钻 > 同环比 > 明细下钻
- 占位符已用真实数据填好，直接选即可

候选池（已按 when 条件过滤）：
{filtered_pool}

用户本轮输入：{user_input}
上轮已问：{last_input}

输出 JSON 数组：["推荐1", "推荐2", ...]
```
